# 목표치 달성 튜닝 순서표

목표:

1. 시스템 성공률 99.5% 이상
2. 품질 성공률 90% 이상

## 1단계: 시스템 안정화 (우선)

1. API 타임아웃/재시도 정책 확인
2. OpenAI/Qdrant 장애 시 fallback 동작 점검
3. 100문항 기준 시스템 실패 케이스(HTTP 비정상/빈 answer) 우선 제거

완료 조건:
- 시스템 성공률 99.5% 이상

## 2단계: Positive 품질 개선

1. 실패 질문을 KB 기준으로 묶어 원인 분류
2. 각 KB에 예상질문 다양 표현 추가
3. 제목을 주제+증상 중심으로 명확화
4. 키워드는 포괄어 제거, 핵심 키워드로 재정리

완료 조건:
- Positive 품질 성공률 92% 이상

## 3단계: Negative 차단 개선

1. 오탐(negative인데 답변 생성) 케이스 수집
2. 임계치/저유사도 안내문/프롬프트 규칙 점검
3. 도메인 외 질문 패턴을 negative 세트에 추가

완료 조건:
- Negative 품질 성공률 80% 이상

## 4단계: 속도 최적화

1. 느린 케이스의 응답시간 로그 분석
2. 상위 병합 후보 수와 문서 보조 검색 조건 점검
3. 필요 시 캐시/요약 길이 조정

완료 조건:
- P95 3000ms 이하

## 5단계: 릴리스 게이트

배포 직전 체크리스트:

1. node scripts/benchmark-100.js 실행
2. reports/benchmark-100-latest.md 확인
3. 목표치 미달 시 배포 보류 후 2~4단계 반복
