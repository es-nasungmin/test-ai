# 운영 성공률 지표 정의서

## 1. 지표 범위

1. 시스템 성공률
- 조건: HTTP 2xx + `answer` 비어있지 않음
- 식: 시스템 성공 건수 / 전체 요청 수

2. 품질 성공률 (100문항 벤치)
- Positive: `isLowSimilarity=false` + 기대 KB 매칭
- Negative: `isLowSimilarity=true`

3. 품질 일관성률 (저장 질문 리플레이)
- 저장된 과거 질문을 재질의했을 때 기존 판단(low/non-low)과 일치하는 비율

4. 응답 지연
- 평균, P50, P95, P99

5. 처리량
- 동시 부하 리플레이에서 req/s

## 2. 권장 목표치 (운영 1차)

1. 시스템 성공률 >= 99.5%
2. 100문항 품질 성공률 >= 90.0%
3. 저장 질문 품질 일관성률 >= 90.0%
4. 지연시간 P95 <= 3000ms
5. 동시성 8 기준 처리량 >= 6 req/s

## 3. 최근 실측 (2026-05-10)

판정:

## 4. 운영 리포트 최소 항목

1. 총 요청 수
2. 시스템 성공률
3. 품질 성공률(100문항)
4. 품질 일관성률(저장 질문 리플레이)
5. 평균/P50/P95/P99 지연
6. 처리량(req/s)
7. 실패 케이스 Top N + 원인 분류

## 5. 주의사항

1. 샘플 수가 작으면 지표가 흔들립니다. 리플레이 라운드로 표본을 확장하세요.
2. `BENCH_CASESET=demo`만으로는 운영 품질을 대표하지 못합니다.
3. 신규 KB 대량 반영 시 벤치 재실행이 필수입니다.
