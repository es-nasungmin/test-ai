일반 HTML Chat 위젯 예시

Vue 없이 순수 HTML + JavaScript만으로 AI 채팅 위젯을 사용할 수 있습니다.
방법 1은 스크립트 한 줄만 추가하면 플로팅 버튼이 자동으로 생기는 가장 간단한 방법이고,
방법 2는 플로팅 버튼 없이 페이지의 기존 버튼에 채팅창을 연결하는 방법입니다.

가장 간단한 사용법

이 코드 그대로 복붙해서 시작

아래 코드를 그대로 붙이면 스크립트 위치와 상관없이 안정적으로 동작합니다.
먼저 이 방식으로 붙여서 확인한 뒤, 필요하면 옵션을 추가하세요.
platformLabel은 인사말에 보여줄 이름이고, platform은 실제 조회에 사용할 플랫폼 값입니다. 

<script src="/chat-widget.js"></script>
<script>
  document.addEventListener('DOMContentLoaded', function () {
    const widget = window.createCrmChatWidget({
      apiBaseUrl: 'http://localhost:8080/api',
      role: 'user',
      userId: 'user-1001',
      username: '홍길동',
      userLoginId: 'gildong@domain.com',
      title: '링크업 고객 챗봇',
      platformLabel: '링크업',
      platform: '링크업',
      buttonLabel: 'Chat',
      themeColor: '#185ca2'
    })

    widget.showWidget()
  })
</script>
1
apiBaseUrl를 실제 백엔드 주소로 맞추세요.
2
스크립트 로드 후 실행되도록 위젯 생성 코드를 배치하세요.
방법 1 · 플로팅 버튼 방식

별도 요소 없이 바로 붙이기

1
스크립트를 페이지에 로드합니다.
<script src="/chat-widget.js"></script>
2
createCrmChatWidget()을 호출하면 끝입니다.
HTML에 아무 요소도 추가할 필요 없이, 플로팅 버튼과 채팅 팝업이 document.body에 자동으로 추가됩니다.
3
필요하면 코드로 제어할 수도 있습니다.
반환된 widget 객체의 메서드를 호출해 버튼·채팅창을 직접 열고 닫을 수 있습니다. 아래 버튼으로 지금 바로 체험해 보세요.
4
platformLabel과 platform을 구분해서 쓰세요.
예를 들어 화면에는 링크업으로 인사하고, API 조회 값은 링크업 또는 트러스트온으로 보낼 수 있습니다. 
// HTML에 추가할 요소 없음 — 스크립트 로드 후 한 줄이면 끝
const widget = window.createCrmChatWidget({
  apiBaseUrl: 'http://localhost:8080/api',
  role: 'user',
  userId: 'user-1001',
  username: '홍길동',
  userLoginId: 'gildong@domain.com',
  title: '링크업 고객 Chat',
  platformLabel: '링크업',
  platform: '링크업',
  buttonLabel: 'Chat',
  themeColor: '#185ca2',
  buttonRight: '20px',   // 플로팅 버튼 우측 여백
  buttonBottom: '20px',  // 플로팅 버튼 하단 여백
  popupRight: '20px',    // 채팅창 우측 여백
  popupBottom: '88px'    // 채팅창 하단 여백
})

// 이후 코드에서 필요할 때 제어 가능
widget.showWidget()  // 플로팅 버튼 다시 표시
widget.hideWidget()  // 플로팅 버튼 숨기기 (채팅창도 닫힘)
widget.openChat()    // 채팅창 바로 열기
widget.closeChat()   // 채팅창 닫기
widget.setUserContext({ userId: 'user-2002', username: 'minsu' }) // 로그인 사용자 변경 반영 (loginId 사용)

위치 조절 체험 (슬라이더를 움직여 보세요)

방법 2 · 내 버튼 연결 방식

기존 버튼에 연결하기

1
스크립트를 페이지에 로드합니다.
<script src="/chat-widget.js"></script>
2
hideButton: true 옵션으로 위젯을 생성합니다.
플로팅 버튼은 만들어지지 않고, 채팅 팝업만 내부적으로 준비됩니다. HTML에 별도 요소를 추가할 필요는 없습니다.
3
페이지에 이미 있는 버튼에 openChat()을 연결합니다.
버튼을 클릭하면 채팅 팝업이 열립니다. 버튼은 어디에 있어도 상관없고, 디자인도 자유롭게 꾸밀 수 있습니다.
4
플랫폼 표시는 platformLabel로 따로 지정할 수 있습니다.
노출 문구와 API 파라미터를 다르게 써야 할 때 platform과 분리해서 설정하세요. 
// 1. 위젯 생성 (플로팅 버튼 없음)
const myWidget = window.createCrmChatWidget({
  apiBaseUrl: 'http://localhost:8080/api',
  role: 'user',
  userId: 'user-1001',
  username: 'gildong',
  title: '링크업 고객 Chat',
  platformLabel: '링크업',
  platform: '링크업',
  buttonLabel: 'Chat',
  themeColor: '#1a7a5e',
  hideButton: true,      // 플로팅 버튼 생성 안 함 (방법 2는 팝업 위치 위주)
  popupRight: '20px',    // 채팅창 위치 직접 지정
  popupBottom: '20px'
})

// 2. 이미 있는 버튼에 연결 (버튼은 직접 HTML에 만들면 됨)
document.getElementById('myBtn')
  .addEventListener('click', () => myWidget.openChat())

// 3. 닫기 버튼도 같은 방식
document.getElementById('closeBtn')
  .addEventListener('click', () => myWidget.closeChat())

채팅창 위치 조절 체험

메서드(함수) 목록

createCrmChatWidget()이 반환하는 widget 객체에서 사용할 수 있는 함수 목록입니다.

showWidget()
플로팅 버튼을 다시 화면에 표시합니다widget.showWidget()
hideWidget()
플로팅 버튼을 숨깁니다. 채팅창이 열려 있어도 함께 닫힙니다widget.hideWidget()
openChat()
채팅 팝업창을 엽니다. 플로팅 버튼 없이 내 버튼으로 열 때 사용widget.openChat()
closeChat()
채팅 팝업창을 닫습니다. 세션은 초기화되지 않습니다widget.closeChat()
setUserContext(ctx)
로그인 상태가 바뀌었을 때 사용자 정보를 업데이트합니다. 이후 전송하는 메시지부터 반영됩니다widget.setUserContext({
userId: 'u-001',
username: '홍길동',
userLoginId: 'gildong'
})
setButtonRight(val)
플로팅 버튼의 오른쪽 여백을 동적으로 변경합니다widget.setButtonRight('40px')
setButtonBottom(val)
플로팅 버튼의 아래쪽 여백을 동적으로 변경합니다widget.setButtonBottom('80px')
setPopupRight(val)
채팅 팝업창의 오른쪽 여백을 동적으로 변경합니다widget.setPopupRight('40px')
setPopupBottom(val)
채팅 팝업창의 아래쪽 여백을 동적으로 변경합니다widget.setPopupBottom('100px')
destroy()
위젯을 DOM에서 완전히 제거합니다. SPA에서 페이지 전환 시 정리할 때 사용widget.destroy()
const widget = window.createCrmChatWidget({ ... })

// 화면 표시/숨김
widget.showWidget()   // 플로팅 버튼 표시
widget.hideWidget()   // 플로팅 버튼 숨기기

// 채팅창 열고 닫기
widget.openChat()     // 채팅 팝업 열기
widget.closeChat()    // 채팅 팝업 닫기

// 로그인 후 사용자 정보 반영
widget.setUserContext({
  userId: 'u-001',
  username: '홍길동',
  userLoginId: 'gildong'   // ← 채팅자 표시에 사용
})

// 위치 조절
widget.setButtonRight('40px')
widget.setButtonBottom('80px')
widget.setPopupRight('40px')
widget.setPopupBottom('108px')

// SPA 페이지 이동 시 정리
widget.destroy()

속성 가이드

apiBaseUrl
백엔드 API 서버 주소. 배포 시 운영서버 URL로 교체string | 기본값: 'http://localhost:8080/api'
role
user: 공개 KB만 검색 (일반 고객용) / admin: 전체 KB 검색 (내부 관리자용)'user' | 'admin' | 기본값: 'user'
platform
KB 관리 페이지에서 등록한 플랫폼 이름과 정확히 일치해야 합니다. GET /api/knowledgebase/platforms 응답값을 그대로 사용하세요. 전체 검색이 필요하면 '전체 플랫폼'을 사용하세요.
⚠ 주의: 등록값이 '링크업'이면 위젯도 '링크업', 등록값이 '트러스트온'이면 위젯도 '트러스트온'으로 정확히 맞춰야 합니다.string | 기본값: '전체 플랫폼'
예: '링크업', '트러스트온', '전체 플랫폼'
platformLabel
인사말·헤더에 표시만 할 이름. 실제 KB 검색에는 사용되지 않습니다.string | 선택값
예: platform='링크업' + platformLabel='링크업'
showPlatformSelector
false: platform 고정 / true: 헤더에 드롭다운 표시, 사용자가 직접 플랫폼 선택boolean | 기본값: false
userId / username / userLoginId
외부 서비스의 사용자 식별값과 로그인 정보를 위젯에 전달. 요청 시 사용자 식별/기록에 사용됨string | 선택값
예: 'user-1001', '홍길동', 'gildong@domain.com'
title
채팅창 헤더에 표시되는 제목string | 기본값: 'AI 상담 어시스턴트'
themeColor
헤더·플로팅 버튼·사용자 말풍선에 적용되는 테마 색상CSS hex string | 기본값: '#1f7a6d'
hideButton
플로팅 버튼을 만들지 않음. 채팅창만 독립 표시하고 openChat()으로 제어boolean | 기본값: false
buttonLabel
우하단 플로팅 버튼 안에 표시할 텍스트string | 기본값: 'CHAT'
popupRight / popupBottom
채팅창 팝업의 우측·하단 여백. 방법 2(내 버튼 연결)에서 우선 조정string (CSS 단위 포함) | 기본값: '20px' / '88px'
buttonRight / buttonBottom
플로팅 버튼의 우측·하단 여백. 방법 1(플로팅 버튼)에서 사용string (CSS 단위 포함) | 기본값: '20px' / '20px'
initiallyOpen
페이지 로드 즉시 채팅창이 열린 상태로 시작boolean | 기본값: false
mountTo
위젯을 삽입할 DOM 요소. CSS 선택자 문자열도 가능Element | string | 기본값: document.body
예: '#chat-area', '.widget-container'
